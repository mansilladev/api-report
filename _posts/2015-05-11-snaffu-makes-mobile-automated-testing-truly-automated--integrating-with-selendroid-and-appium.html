Snaffu, an in-app bug submission tool launched by Clearbridge Mobile, recently released a set of APIs that allow integration with test automation frameworks like Selendroid and Appium. The introduction of these APIs will extend automation testing capabilities for development and QA teams.

With so many mobile devices and environments to test on, many development teams have opted to run automated testing scripts to save time. However, these tools still involve significant manual effort. The Snaffu APIs eliminate this need. If a test case fails during automated testing, Snaffu is automatically triggered capturing relevant information like screen shots, device details/logs, network states and more. Snaffu then pushes the issue to tracking tools like Jira, Pivotal Tracker, and Asana. Users no longer have to manually confirm and submit any issues found through automation testing scripts.

“What we’ve done is make automated testing a truly automated process,” said Deepak Chopra, CEO of Clearbridge Mobile. “These APIs will let teams save time on investigating and logging issues because everything is done automatically via Snaffu.”

“Teams can now run their automation scripts and have them not only identify bugs, but also log them,” added Clearbridge CTO Sanjay Malhotra. “Users don’t even have to hit a button to submit issues.”

The Snaffu team is currently working on extending the APIs to other mobile test automation frameworks like Calaba.sh.
