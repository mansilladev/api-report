Until now we’ve recommended using HTTPS when accessing the Phaxio API, but it hasn’t been exclusively enforced. While all official Phaxio clients are already configured to access the API via HTTPS, some third party libraries may not. Please make sure to to check your codebase to verify that your Phaxio API endpoint is "https://api.phaxio.com", as API calls not sent via HTTPS will begin to fail in the coming weeks.

If you have any questions, please reach out to us at support@phaxio.com.
