This past September, Runscope released Radar Agent, an on-premises testing agent for running tests against APIs and backend services behind the firewall. Today, Runscope has released the Runscope Gateway Agent, an on-premises gateway based on the same software that powers Runscope's cloud infrastructure for making and logging API calls. The Runscope Gateway Agent can now be downloaded and installed on a host within a local network. Deploying the Runscope Gateway Agent within an infrastructure will help to:

  • Capture API request and response data made to private or internal APIs that reside behind a firewall
  • Troubleshoot and debug those API calls using the Runscope Traffic Inspector
  • Increase network performance by keeping API traffic within an infrastructure
  • Create Runscope URLs (on a custom domain name) for services running on localhost
  • Access APIs that require a fixed client IP address (IP whitelisting)

The agent works by creating a locally-hosted instance of the Runscope Gateway with a custom hostname and TLS certificate. Instead of using Runscope's cloud-based gateways (hosted on *.runscope.net), requests will use the configured hostname for the agent instead. For complete details, see the Runscope Gateway Agent documentation.
